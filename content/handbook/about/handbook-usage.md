## Handbook First

The handbook is the preferred source of durable operating knowledge.

If a decision, process, standard, or practice should survive beyond a meeting or
chat thread, it belongs in the handbook.

## Public First

Public operating knowledge should be published in the public handbook whenever
it can be shared safely.

Private material should not be hidden in random documents. When private
handbook content becomes necessary, it should live in a separate private
handbook with the same contribution discipline.

## Change Flow

1. A process or knowledge gap appears.
2. A contributor proposes a handbook change in a pull request.
3. The relevant maintainers review it through `CODEOWNERS`.
4. The change is merged.
5. Teams link to the merged diff or handbook page as the source of truth.

## Single Source of Truth

Avoid duplicating handbook content in slide decks, chat messages, pages, or
documents. Link to the handbook instead.

If duplicate content exists, replace one copy with a link to the canonical
handbook page.
