## Write Plainly

Use direct language. Prefer short sections and descriptive headings.

## Name the Real Thing

Avoid vague page titles like "Resources" or "Process." Name the function,
result, standard, or decision the page owns.

## Preserve Context

When adding a process, include the reason behind it when the reason helps future
contributors decide whether it should change.

## Link Instead of Duplicating

If another page already owns the idea, link to it instead of repeating it.
