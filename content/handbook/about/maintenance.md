## Maintainers

Page maintainers are derived from `.github/CODEOWNERS`.

Do not add maintainer lists to individual pages unless there is a temporary
reason that cannot be represented through repository ownership.

## Freshness

Git history provides the last updated date and contributor history.

For sensitive pages, maintainers may add a visible review cadence later through
centralized generated data rather than ad hoc page footers.

## Broken Structure

If a page grows too large, split it by function or result. If two pages repeat
the same idea, choose a single source of truth and replace the duplicate with a
link.
